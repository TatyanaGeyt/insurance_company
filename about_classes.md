# Описание классов и структур (по текущему коду)

Этот файл — расшифровка сущностей проекта в соответствии с текущим кодом (`my_classes.h/.cpp`).

## `InsuranceType` (enum class)
Три фиксированных вида страховки:
- `Home` — жильё
- `Car` — авто
- `Health` — здоровье

Используется как индекс (0..2) во всех массивах симуляции.

## `Offer` (class)
**Назначение:** актуальные условия, по которым компания продаёт новые договоры определённого типа.

### Ключевой смысл
- В симуляции есть **ровно одно актуальное предложение** на каждый тип страхования (`Simulation::offers_[3]`).
- Проданные договоры не меняются при изменении оффера: условия фиксируются “снимком” (`Contract::offerSnapshot`).

### Поля (все приватные, доступ к ним осуществляется через getter/setter)
- `type_ : InsuranceType` — тип предложения (задаётся симуляцией).
- `premiumPerMonth_ : double` — ежемесячный взнос.
- `contractMonths_ : int` — длительность договора в месяцах.
- `maxCoverage_ : double` — максимальная сумма ущерба (верхняя граница `damage`).
- `deductible_ : double` — франшиза-порог: выплаты начинаются при `damage >= deductible`.
- `offerValidMonths_ : int` — срок действия условий предложения (в месяцах); **уменьшается каждый шаг**, при достижении 0 шаг блокируется, пока менеджер не обновит оффер (вспылвает окно предупреждения).
- `baseDemand_ : int` — базовый спрос (задаётся из `Config::baseDemand` - настройка системы).
- `curDemand_ : double` — “чистый” текущий спрос до шума (пересчитывается каждый месяц, не как случайная величина; случайность добавляется с помощью шума).

### Методы
- `get*/set*` — доступ к приватным полям.
- `totalPrice()` — стоимость договора для формулы спроса: `premiumPerMonth * contractMonths`.

## `Contract` (struct)
**Назначение:** один заключённый договор.

### Поля
- `offerSnapshot : Offer` — снимок условий при покупке.
- `startMonth : int` — месяц начала (1..M).
- `endMonth : int` — месяц окончания (включительно).

### Методы
- `isActive(month)` — активен ли договор в заданном месяце.
- `type()` — тип договора через `offerSnapshot.getType()`.

## `Config` (struct)
**Назначение:** параметры “мира” симуляции, которые задаются в стартовом диалоге.

### Поля
- `M` — число месяцев моделирования.
- `initialCapital` — начальный капитал.
- `taxRate` — налог на капитал (доля 0..1), списывается **каждый месяц** с текущего капитала.
- `baseDemand[3]` — базовый спрос по типам.
- `demandNoiseMax` — шум спроса (добавка 0..N покупателей).
- `minClaimsPerMonth/maxClaimsPerMonth` — диапазон числа страховых случаев в месяц (равномерно) **на тип** (для каждого типа одинаковый диапазон), но ограничивается сверху числом активных договоров этого типа.
- `seed` — seed для генератора случайных чисел (0 = случайный).

## `MonthResult` (struct)
**Назначение:** “отчёт” за один месяц для UI-таблицы.

### Поля
- `month` — номер месяца.
- `capitalBefore`, `capitalAfter` — капитал до/после месяца.
- `taxPaid` — налог за месяц.
- `sold[3]` — сколько договоров продано в этом месяце по типам.
- `revenue[3]` — доходы по типам за месяц.
- `claimsCount[3]` — число страховых случаев по типам.
- `claimsPaid[3]` — выплаты по типам.
- `totalClaims` — сумма выплат за месяц.
- `effectiveDemand[3]` — рассчитанный спрос (до добавления шума).

## `Simulation` (class)
**Назначение:** основной класс игры: хранит капитал, предложения, договоры и выполняет месячный шаг.

### Хранимые данные
- `cfg_ : Config` — настройки.
- `offers_[3] : Offer` — актуальные офферы по типам.
- `offerArchive_[3] : vector<Offer>` — архив всех офферов, которые когда-либо были актуальными.
- `offerArchiveContractsSold_[3] : vector<int>` — число заключённых договоров для каждого элемента архива.
- `contracts_ : vector<Contract>` — все договоры (каждый месяц истёкшие удаляются).
- `month_`, `capital_`, `bankrupt_`, `finished_`, `rng_`.

### Основные методы
- `reset()` — сбрасывает месяц/капитал/договоры/архивы и синхронизирует `baseDemand` в офферах.
- `setConfig(const Config&)` — применяет настройки и вызывает `reset()`.
- `setOffer(InsuranceType, const Offer&)` — меняет актуальный оффер и добавляет его в архив; снимает “истечение” для типа.
- `computeDemand(const Offer&)` — текущий спрос по правилу \(mult = maxCoverage/totalPrice\) (без “потолка”, который был изначально).
- `activeContractsByType(month)` — количество активных договоров по типам для заданного месяца (используется UI).
- `needsOfferUpdate()`/`expiredOffers()` — проверка, истекли ли условия и нужен ли менеджер (по `offerValidMonths == 0`).
- `step(MonthResult*)` — шаг в один месяц:
  1) списание налога `taxRate * capital`
  2) продажи: `sold ≈ round(curDemand) + noise`
  3) доход за месяц: **помесячный** — сумма `offerSnapshot.premiumPerMonth` по всем активным договорам в текущем месяце (каждый договор платит по “снимку” условий)
  4) страховые случаи: число случаев генерируется на тип и ограничивается сверху количеством активных договоров этого типа; по каждому случаю генерируется ущерб `damage = maxCoverage * u`, и если `damage >= deductible`, выплачивается `damage`
  5) банкротство: если `capital < totalClaims`, выставляется `bankrupt_` и шаг возвращает `false`
  6) истечение условий оффера: `offerValidMonths` уменьшается; при 0 `needsOfferUpdate()` становится `true` и дальнейшие шаги блокируются, пока менеджер не обновит оффер.

## Краткие связи
- `Simulation` агрегирует `Offer`, `Contract`, `Config`.
- `Contract` композиционно хранит `Offer` как снимок.
- `MonthResult` — DTO-структура для UI.
